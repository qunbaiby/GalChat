# GalChat 剧情编辑器操作手册

> 适用对象：剧情策划、文案策划、关卡策划、内容验收人员  
> 适用版本：Godot 4.6.3 / 当前 `addons/story_editor` 插件  
> 文档目的：不直接编辑 JSON，也能完成剧情创建、编排、校验、保存和联调。

---

## 1. 开始之前

### 1.1 编辑器能处理什么

剧情编辑器目前统一管理以下内容：

| 内容 | 用途 | 在哪里打开 |
|---|---|---|
| 主线剧情 | 固定章节、对白、演出、选项和章节分支 | 左侧“主线剧情” |
| 地图世界 | 在地图或世界条件下触发的固定剧情 | 左侧“地图世界” |
| 手机消息 | 固定手机聊天、玩家回复和消息跳转 | 左侧“手机消息” |
| 固定来电 | 角色来电、顺序台词和 TTS 文本 | 左侧“固定来电” |
| AI 约会 | 约会模板、生成参数、Prompt、候选生成和审核 | “内容库 > AI 约会” |
| 手机 AI | 手机文字和 AI 通话的 Prompt、响应测试 | “内容库 > 手机 AI” |
| 剧情入口 | 日程、地图、Guide Flow、Event Registry 引用 | “内容库”菜单 |
| 运行检查 | 校验、分支模拟、覆盖率、运行时事件 | “剧情工具”“系统工具” |

### 1.2 打开编辑器

1. 启动 Godot 编辑器并打开 GalChat 项目。
2. 在 Godot 顶部菜单选择 **工具 > GalChat 剧情编辑器**。
3. 剧情编辑器以独立窗口打开，建议使用不低于 $1100\times700$ 的窗口尺寸。

关闭窗口通常只是隐藏窗口。再次从“工具”菜单打开时，尚未保存的主窗口内容会继续保留；但不要把关闭窗口当成保存。

### 1.3 最重要的三个概念

编辑器中存在三种不同的数据动作：

| 标记 | 含义 | 常见操作 |
|---|---|---|
| **即时更新** | 操作立即进入编辑器内存，但尚未写入文件 | 新增、删除、移动节点；改变连线 |
| **需要应用** | 表单内容只有点击“应用”后才进入编辑器内存 | Inspector 字段、通话信息、完成动作 |
| **直接写盘** | 确认后立即修改项目文件 | 新建固定内容、Schema 迁移、保存模板 |

请记住：

> **应用不等于保存。** 先应用表单，再点击对应编辑器的“保存”，文件才真正更新。

当主剧情标题后出现 `*`，或状态显示“未保存”，表示内存中的内容还没有写回磁盘。

## 2. 主窗口速览

主窗口从上到下分为五个区域。

### 2.1 顶部文件栏

| 控件 | 作用 |
|---|---|
| 当前文档标题 | 显示正在编辑的内容；`*` 表示有未保存修改 |
| 刷新 | 重新扫描项目内的剧情和固定内容 |
| 保存剧情 | 保存当前主线或地图剧情 JSON |
| 撤销 / 重做 | 撤销或恢复当前工作区的编辑 |

### 2.2 工具菜单

- **剧情工具**：校验当前剧情、分支模拟、节点模板库、覆盖率报告。
- **内容库**：AI 约会、手机 AI、剧情引用、Guide Flow、入口调度。
- **系统工具**：Schema 迁移、运行时监视。

### 2.3 左侧固定内容库

左侧按以下分类展示内容：

1. 主线剧情
2. 地图世界
3. 手机消息
4. 固定来电

搜索框支持按 ID、名称、角色和路径筛选。点击分类标题不会打开文档，需要点击分类下的具体内容。

### 2.4 中央工作区

编辑主线或地图剧情时，中央区域依次显示：

1. 章节选择与章节管理。
2. 事件搜索。
3. 事件新增、复制、移动和模板工具。
4. 节点画布。
5. 自动校验结果。

编辑手机消息或固定来电时，中央区域会切换为对应的内嵌编辑器。

### 2.5 右侧 Inspector

选择主线或地图剧情中的事件节点后，可在右侧编辑：

- **结构化**：策划日常使用的字段表单。
- **高级 JSON**：处理尚未提供表单的扩展字段。
- **应用到当前剧情**：把结构化表单写入当前剧情内存。
- **应用高级 JSON**：把完整 JSON 对象写入当前事件内存。

日常编辑优先使用“结构化”。除非明确知道运行时协议，否则不要修改高级 JSON。

## 3. 十分钟上手：创建一段主线剧情

下面是一条最短、最安全的完整流程。

### 3.1 新建剧情

1. 点击左侧 **+ 新建固定内容**。
2. 类型选择 **主线剧情**。
3. 填写内容 ID，例如 `luna_chapter_01`。
4. 填写显示名称，例如“Luna 第一章”。
5. 确认保存位置后点击确认。

新建成功后，文件会立即写入磁盘并自动打开。新剧情默认包含 `start` 章节和一条旁白。

内容 ID 规则：

- 只能使用英文字母、数字、下划线 `_` 和连字符 `-`。
- 不能包含空格、点号或中文。
- 必须在对应内容类型内唯一。
- 推荐使用“角色_章节_序号”，例如 `luna_main_01`。

### 3.2 编辑默认对白

1. 点击画布中的默认“对白”节点。
2. 在右侧“结构化”页设置发言者、表情和对白内容。
3. 点击 **应用到当前剧情**。
4. 确认标题出现 `*`。

### 3.3 添加场景演出

1. 在事件类型下拉框选择 **时段卡片**，点击 **新增事件**。
2. 填写背景、时段、地点名和停留时长，点击应用。
3. 选择 **切换背景**，新增并设置背景与过渡方式。
4. 选择 **显示角色**，设置角色、位置、表情和动画。
5. 继续添加对白。

也可以直接选择 **组合模板 > 地点与时段开场**，一次插入一组常用节点，再逐个补齐字段。

### 3.4 添加玩家选择

1. 选择事件类型 **玩家选项**，点击新增。
2. 在右侧为每个选项填写唯一 ID、显示文本、数值效果和目标章节。
3. 点击 **新增选项** 可增加更多选项。
4. 点击 **应用到当前剧情**。

若选项不跳转到其他章节，可以不设置目标章节，让剧情按数组顺序继续。若设置了目标章节，画布会显示分支线。

### 3.5 收尾并保存

1. 新增 **隐藏角色**，让角色退出画面。
2. 新增 **背景音乐** 或 **音效**，动作选择停止。
3. 新增 **跳转章节**，目标选择“剧情结束”。
4. 查看下方“校验结果”，修复所有错误。
5. 点击顶部 **保存剧情**。

保存后标题中的 `*` 应消失，状态应回到“已保存”。

## 4. 创建四类固定内容

点击左侧 **+ 新建固定内容**，可选择以下类型。

### 4.1 主线剧情

- 必填：内容 ID。
- 可选：显示名称，用于初始剧情摘要。
- 保存位置：`assets/data/story/scripts/main/`。
- 初始结构：`start` 章节和一条旁白。

### 4.2 地图世界

- 必填：内容 ID。
- 可选：显示名称。
- 保存位置：`assets/data/story/scripts/events/`。
- 编辑方式与主线剧情完全相同。

地图剧情文件创建后，还需要通过“剧情引用”或“入口调度”配置触发入口，否则它可能永远不会在游戏中出现。

### 4.3 手机消息

- 必填：内容 ID、角色 ID。
- 保存位置：`assets/data/mobile/fixed_chats/`。
- 初始结构：一条 ID 为 `m1` 的普通消息。

角色 ID 必须使用项目中的真实角色 ID。创建成功后，从左侧“手机消息”分类打开并继续编辑。

### 4.4 固定来电

- 必填：内容 ID、角色 ID。
- 保存位置：共享固定来电数组。
- 初始结构：一行“新通话台词”。

固定来电不是单独文件。新建后会直接加入共享数组，因此创建前要确认 ID 和角色无误。

### 4.5 新建操作的注意事项

- 新建固定内容属于**直接写盘**。
- 编辑器不会覆盖同名内容。
- 创建成功后仍需完成内容编辑、校验和保存。
- 创建错类型时不要通过高级 JSON 强行转换，应删除错误资源后按正确类型重建。

## 5. 主线与地图剧情编辑

### 5.1 打开和切换剧情

1. 在左侧选择“主线剧情”或“地图世界”。
2. 点击具体剧情。
3. 中央画布加载 `start` 章节，右侧等待选择事件。

当前剧情存在未保存修改时，编辑器会阻止直接切换到另一段主线或地图剧情。先保存或撤销修改，再切换。

### 5.2 管理章节

| 操作 | 使用方式 | 注意事项 |
|---|---|---|
| 选择章节 | 使用“章节”下拉框 | 新剧情默认从 `start` 开始 |
| 新增章节 | 点击“新增章节”并输入章节 ID | ID 必须唯一 |
| 改名 | 选择章节后点击“改名” | Jump 和 Choice 目标会同步更新 |
| 删除章节 | 选择章节后点击“删除章节” | 会删除该章节全部事件 |
| 进入分支章节 | 双击画布中的章节入口 | 可快速切换目标章节 |

至少保留一个章节。推荐始终保留 `start` 作为剧情入口。

### 5.3 新增和编排事件

1. 从事件类型下拉框选择类型。
2. 点击 **新增事件**。
3. 若已选择事件，新事件插入在当前事件之后；否则插入到章节末尾。
4. 选择新节点，在 Inspector 填写字段并应用。

事件栏还提供：

- **复制 / 粘贴**：复制事件及全部扩展字段。
- **重复**：在当前事件后创建完整副本。
- **上移 / 下移**：调整单个事件的播放顺序。
- **删除事件**：删除当前选择。
- **定位选中**：把所选节点移动到画布中央。
- **整理画布**：按流程自动排列节点。

连续选择多个事件后，可以成组复制、粘贴、重复或删除。上移和下移只适用于单个事件。

快捷键：

| 快捷键 | 操作 |
|---|---|
| `Ctrl+C` | 复制选中事件 |
| `Ctrl+V` | 粘贴到当前事件之后 |
| `Ctrl+D` | 重复选中事件 |
| `Ctrl+Z` | 撤销 |
| `Ctrl+Y` 或 `Ctrl+Shift+Z` | 重做 |

当输入框正在输入文字时，编辑器不会抢占这些快捷键。

### 5.4 理解画布连线

- **顺序线**：表示事件数组的先后播放顺序，通常不需要手工连接。
- **分支线**：表示 Choice 或 Jump 到目标章节的跳转。
- **章节入口**：代表某个章节或“剧情结束”。
- **双击入口**：进入该章节。

拖动节点只改变编辑器布局，不改变播放顺序。要改变播放顺序，请使用“上移 / 下移”。

### 5.5 搜索事件

“事件搜索”可搜索：

- 章节 ID 和事件序号。
- 事件类型。
- 角色和发言者。
- 对白文本。
- 背景、音频、通话等资源 ID。
- Choice 选项。
- 高级 JSON 中的扩展字段。

输入关键词后，使用“上一个”“下一个”循环定位。按 `Enter` 等同于“下一个”。

### 5.6 组合模板

内置模板包括：

| 模板 | 插入内容 |
|---|---|
| 地点与时段开场 | 时段卡片、背景、BGM、旁白 |
| 双人会面 | 角色入场、移动、双方出现和对白 |
| 回应型玩家选择 | 引导对白、Choice、后续对白 |
| 固定剧情 AI 插曲 | 前置对白、AI 对话、恢复对白 |
| 引导式 AI 主线 | 前置旁白、受剧情约束的限轮 AI 对话、后续旁白 |
| 章节收束与结算 | 退场、停止音频、设置变量、结束剧情 |

模板插入后仍需逐个填写资源 ID、角色、文本等字段。

要保存自己的组合模板：

1. 在画布中连续选择一组事件。
2. 打开 **组合模板 > 保存选中事件组合为模板**。
3. 输入模板名称并确认。

自定义模板保存在项目模板库，其他策划也可以使用。删除模板不会删除已经插入剧情的事件。

## 6. 事件类型说明

### 6.1 对白

用于角色发言或旁白。

| 字段 | 说明 |
|---|---|
| 发言者 | 选择旁白、玩家或角色 |
| 角色 ID | 与立绘、表情等资源关联的角色 |
| 表情 | 从该角色已注册表情中选择 |
| 显示名 | 可选，覆盖默认角色名 |
| 情绪 | 当前对白的情绪标签 |
| 聚焦 | 是否突出当前发言角色 |
| 语音指令（TTS 2.0） | 可选，逐句控制情绪、语速、语气或音量，最多 80 个字符 |
| 对白内容 | 实际显示文本 |

动作描述建议放在对白开头，并使用全角括号，例如：`（看向窗外）今天的雨还没有停。`

语音合成统一使用豆包 Seed-TTS 2.0。通常只需选择“表情”，系统会自动生成克制的默认语音指令；需要精细演出时再填写“语音指令（TTS 2.0）”，例如：`略带害羞地慢一点说`、`稍微压低音量，认真地说`。指令只应描述当前句的说话方式，不要要求改变年龄、性别、身份或音色，例如不要填写“换成萝莉音”“用御姐声说”。系统会自动追加保持原本声线的约束。

### 6.2 切换背景

- 背景 ID：从已注册图片资源选择。
- 过渡类型：例如 `fade`、`dissolve`、`wipe_left`。
- 过渡时长：单位为秒；大于 0 时会等待过渡完成。

### 6.3 音效

- 音频类型：`bgm`、`bgs` 或 `se`。
- 动作：`play`、`stop` 或 `switch`。
- 淡入淡出：单位为秒。
- 循环：环境音和持续音乐通常启用，一次性音效通常关闭。

### 6.4 背景音乐

专门控制 BGM。新内容优先选择“音频 ID”，不要填写“兼容路径”；兼容路径仅用于旧数据。

### 6.5 显示角色

让角色进入画面。设置角色、位置、表情、入场动画、显示名和聚焦状态。

### 6.6 移动角色

调整已经显示的角色位置、表情和视觉焦点。不要用它代替首次入场。

### 6.7 隐藏角色

让指定角色退场。章节结束前建议清理不再需要的角色。

### 6.8 时段卡片

用于地点或时间切换：

- 背景 ID：卡片背景。
- 时段：例如上午、下午、夜晚。
- 地点名：玩家看到的地点名称。
- 停留时长：卡片展示秒数。

### 6.9 玩家选项

每个选项包含：

- 选项 ID：同一 Choice 内唯一。
- 选项文本：玩家看到的内容。
- 目标章节：可选；设置后跳转到该章节。
- 亲密度效果。
- 信任度效果。

建议每条分支都明确决定：

1. 跳转到独立章节；或
2. 不设置目标，顺序回到共同后续。

不要让某些选项跳转、另一些选项无意中顺序继续。

### 6.10 跳转章节

结束当前顺序流并跳到指定章节。选择“剧情结束”表示终止剧情。

### 6.11 设置变量

- 变量名：供运行时和后续流程使用。
- 变量值：必须是合法 JSON 值，例如 `true`、`3`、`"completed"` 或数组/对象。

变量名和用途应先在策划表中约定，避免不同剧情使用近似但不一致的名称。

### 6.12 AI 对话

暂停固定剧情并启动受 Prompt 控制的临时 AI 对话。玩家结束 AI 对话后，固定剧情继续。

Prompt 应明确：

- 本次对话目标。
- 当前角色和关系边界。
- 必须围绕的话题。
- 禁止偏离的内容。
- 何时自然结束。

### 6.13 引导式 AI 主线对话

用于需要玩家参与、但不能无限聊天或改写关键事实的主线片段。剧情锚点和场景目标必填；每个必达剧情点都需要唯一 ID 和明确的表达指令。必达剧情点数量不能超过最大玩家回合。

- 允许讨论范围：角色可以自然回应的话题边界。
- 禁止改写事实：已经发生且不得被 AI 否认、替换或新增的事实。
- 偏题回拉策略：玩家偏题时如何先回应再拉回主线。
- 隐藏主动结束：主线建议开启，由角色在达到限制后自然收束。
- 允许提前收束：所有必达剧情点完成后，不再等待剩余回合。
- 结果章节映射：仅支持 `complete` 和 `incomplete`，值为章节 ID 或 `end`。
- 时间与行动力：行动力在进入前扣除；未能进入时不会推进剧情时间。

运行时按回合提供候选必达剧情点。AI 在同一次结构化响应中返回角色台词、已覆盖的剧情点 ID 和台词证据；只有证据确实逐字出现在角色台词中，该剧情点才会记为完成。收束阶段会再次要求覆盖尚未完成的剧情点，最终根据实际覆盖情况进入 `complete` 或 `incomplete`。

必达剧情点适合描述角色需要明确表达的信息，不适合判断玩家的真实态度、承诺或情绪。需要依据玩家选择进行分支时，应使用固定 Choice 或独立条件事件。

### 6.14 自由聊天

进入限定轮数的自由交流：

- 对话策略：本次交流的主题和边界。
- 最大轮数：允许玩家交流的轮数。

### 6.15 语音通话

通过固定通话 ID 播放已创建的来电。先在左侧“固定来电”中创建并保存通话，再回到事件中选择它。

### 6.16 称呼设置弹窗

请求玩家设置角色对自己的称呼。该事件没有额外字段。

### 6.17 高级 JSON

高级 JSON 会编辑当前事件的完整对象，适合处理未建模扩展字段。

操作顺序：

1. 选择事件。
2. 打开“高级 JSON”。
3. 修改后点击 **应用高级 JSON**。
4. 查看校验结果。
5. 保存剧情。

根值必须是 JSON 对象。高级 JSON 会整体应用，错误字段可能导致运行时跳过事件，仅建议熟悉协议的人员使用。

## 7. 手机固定消息

### 7.1 打开消息

1. 在左侧展开“手机消息”。
2. 选择具体聊天资源。
3. 中央显示消息画布、消息 Inspector、完成动作和校验结果。

画布中：

- 灰色线表示消息数组顺序。
- 绿色线表示玩家选项的 `next` 跳转。

### 7.2 编辑普通消息

1. 选择消息节点。
2. 编辑消息 ID、发送者、文本、图片 ID、延迟、语音标记和语音时长。
3. 点击 **应用消息修改**。

普通消息至少需要文本或图片。消息 ID 在当前聊天中必须唯一。

### 7.3 编辑玩家选项

将发送者设为 `player_options` 后，Inspector 会切换为玩家选项模式。每个选项填写：

- 选项 ID。
- 显示文本。
- 下一消息 ID。

也可以在画布中连接绿色分支线设置下一消息。目标消息必须存在。

### 7.4 调整消息流程

- **新增**：在当前消息后新增消息。
- **删除**：删除当前消息。
- **上移 / 下移**：调整数组顺序。
- **整理画布**：重新排列消息节点。
- 拖动节点：只改变画布位置。

新增、删除、移动和连线属于即时更新；Inspector 表单必须点击应用。

### 7.5 完成动作

“完成动作”是聊天结束后执行的 JSON 数组，目前常用动作包括：

- `activate_goal`：激活目标。
- `activate_main_chat_topic`：激活主界面聊天话题。

当完成动作中存在 `activate_main_chat_topic` 时，可以直接使用完成动作上方的两个选择器：

1. 在“主线话题动作”中选择要配置的动作。
2. 在“后续主线”中选择固定微聊关闭后自动触发的主线剧情。
3. 使用清除按钮可以移除绑定，恢复到 `topic_prompt_hint` 驱动的话题 AI。

主线选择器只显示“主线剧情”目录中的脚本。若已绑定的脚本被删除，编辑器会保留原路径并显示缺失提示，不会静默改写数据。原始 JSON 编辑框仍用于 `auto_start_source_type`、`expire_on_next_day` 等高级字段；直接修改 JSON 后必须点击 **应用动作**。

### 7.6 校验和保存

点击 **保存** 时会再次校验。以下问题会阻止保存：

- 消息 ID 重复。
- `next` 指向不存在的消息。
- 无出口循环。
- 普通消息没有文本和图片。
- 玩家选项为空。
- 语音时长无效。
- 完成动作类型不受支持。

缺少 `next` 可能只产生警告，并按数组顺序继续；仍建议明确设置每个玩家选项的去向。

## 8. 固定来电

### 8.1 编辑基本信息

1. 在左侧“固定来电”选择通话。
2. 修改通话 ID 和来电角色。
3. 点击 **应用通话信息**。

通话 ID 必须唯一。修改已经被剧情引用的通话 ID 后，要同步更新所有“语音通话”事件。

### 8.2 编辑台词

1. 点击 **新增台词** 添加一行。
2. 直接编辑各行文本。
3. 使用每行的上移、下移、删除按钮调整顺序。
4. 点击 **应用台词修改**。

播放规则：

- 台词按照数组顺序自动播放。
- `[SPLIT]` 可在一行内拆分消息块。
- 括号中的舞台说明不会发送给 TTS。
- 至少保留一行非空台词。

### 8.3 保存与共享文件风险

固定来电共用一个数组文件。点击保存时会校验全部通话，任意一条通话有阻塞错误都可能导致保存失败。

保存前检查：

- 通话 ID 是否重复。
- 角色是否存在。
- 是否至少有一行有效台词。
- 剧情中的 `call_id` 是否都能找到。

“重新加载”会重新读取磁盘内容。存在未保存修改时不要点击。

## 9. 剧情工具

### 9.1 校验当前剧情

路径：**剧情工具 > 校验当前剧情**。

编辑器在加载、应用和结构变化后也会自动校验。诊断通常分为：

- **错误**：结构损坏、目标不存在、必填字段缺失等，应在提交前全部修复。
- **警告**：内容可能可运行，但存在不可达、未连接或体验风险。
- **信息**：补充说明。

双击诊断项可定位对应章节或事件。

主剧情目前不会因为存在校验错误而强制禁止保存，因此“保存成功”不代表“内容正确”。提交前必须主动清零错误。

### 9.2 分支模拟

路径：**剧情工具 > 分支模拟**。

1. 在“初始变量 JSON”输入模拟起始状态，例如 `{"relationship_stage": 2}`。
2. 点击 **运行**。
3. 查看每条路径的结束状态、Choice 链、累计效果、最终变量和步数。
4. 双击结果可定位最后执行事件。

模拟器目前执行：

- `set_variable`。
- Choice 数值效果。
- Choice 目标章节。
- Jump。

它不能代替游戏内完整联调，尤其不能模拟尚未定义的 Condition 事件、UI 动画、音频和 AI 服务。

### 9.3 节点模板库

路径：**剧情工具 > 节点模板库**。

模板可按名称、说明和类型搜索。选择模板后：

1. 查看模板说明和载荷预览。
2. 填写需要的参数 JSON。
3. 点击插入当前章节。
4. 回到主窗口补齐字段并保存剧情。

模板插入只修改当前剧情内存。模板重命名或删除会直接修改项目模板库。

### 9.4 覆盖率报告

路径：**剧情工具 > 覆盖率报告**。

报告用于检查：

- 事件结构可达率。
- 分支模拟覆盖率。
- 运行时实际命中。
- 缺失或未使用资源。
- 仓库级剧情诊断。

勾选“合并运行时”时，需要先运行游戏并产生调试 Session。双击事件或资源引用可返回主编辑器定位。

## 10. 内容库工具

### 10.1 剧情引用

路径：**内容库 > 剧情引用**。

用于查看哪些系统会触发某段剧情，包括：

- Event Registry。
- Guide Flow。
- 剧情日程。
- 地图入口。

只有 Event Registry 项可在此直接编辑。流程为：

1. 选择引用项。
2. 修改事件 ID、触发方式、目标路径、可重复和条件。
3. 点击 **应用修改**。
4. 确认没有阻塞错误。
5. 点击 **保存 Registry**。

### 10.2 Guide Flow

路径：**内容库 > Guide Flow**。

支持步骤类型：

- `message`：显示引导消息。
- `wait_action`：等待玩家完成动作。
- `play_story`：播放指定剧情。

编辑流程：

1. 选择 Guide 和步骤。
2. 设置步骤 ID、类型、标题和显示文本。
3. 普通步骤填写等待动作和目标场景。
4. `play_story` 填写剧情路径、Script ID 和结束后是否返回主场景。
5. 点击 **应用步骤修改**。
6. 点击 **保存 Guide**。

### 10.3 入口调度

路径：**内容库 > 入口调度**。

该工具包含三页。

#### 剧情日程

按 Day Offset 配置全天、上午、下午、傍晚和夜晚会触发的剧情事件 ID。修改后点击 **应用日程修改**。

#### 地图入口

可配置：

- 入口 ID 和剧情路径。
- Day Offsets。
- 激活事件。
- 天气和时段。
- 关系阶段上下限。
- 优先级。
- 地图徽标 NPC 和文字。

修改后点击对应的应用按钮。

#### 触发模拟

输入日期、时段、天气、地点、关系阶段、属性和已触发事件，查看每个候选入口是否激活以及失败原因。

全部修改完成后点击 **保存两个数据源**。该操作依次保存两个文件；若第二个文件保存失败，应立即根据错误修复并重试，避免两个数据源版本不一致。

## 11. AI 约会工作台

路径：**内容库 > AI 约会**。

### 11.1 推荐工作流

工作台按四步组织：

1. 选择模板与参数。
2. 生成候选。
3. 审核与比较。
4. 保存可复用模板。

### 11.2 选择或创建模板

左侧可搜索通用类型模板和地点专属模板。选择模板后先阅读用途、约束和必须桥段。

点击 **+ 新建** 可创建：

- 通用约会类型模板。
- 地点专属模板。

新模板 ID 必须唯一。创建模板会直接修改模板配置文件。

### 11.3 填写生成输入

先设置客观环境：

- 地点 ID。
- 时段。
- 天气 ID 和天气描述。
- 关系阶段。
- 亲密度和信任度。
- 创意种子。

再填写本次创意约束：

- 共同任务。
- 微小意外。
- 谈话主题。
- 收束方式。

这些字段应具体、可演出、符合当前关系阶段。不要只写“甜蜜互动”“发生有趣的事”等泛化描述。

### 11.4 离线预览并审核

1. 在“原始响应”中放入待检查的模型响应 JSON。
2. 点击 **预览并审核**。
3. 依次查看：
   - 质量总览。
   - Prompt 预览。
   - 清洗后结果。
   - Fallback。
   - 审核项。

重点检查：

- speaker 是否合法。
- 是否出现 Markdown 或 BBCode。
- 对白数量是否足够。
- 是否存在模板化、泛化表达。
- 内容是否越过关系阶段边界。
- 清洗后事件是否仍然连贯。

Fallback 只是保证存在可运行结果，不代表内容通过策划审核。

### 11.5 真实生成候选

1. 设置生成数量，范围为 1 到 20。
2. 设置失败重试，范围为 0 到 3。
3. 点击 **生成候选**。
4. 根据需要暂停、继续或取消队列。
5. 生成结束后点击 **审核候选** 比较重复度和质量。

真实生成会调用外部 AI 服务并消耗 API 配额。队列会记录耗时、Token、模型、HTTP 状态和响应 ID。

候选结果只保存在当前工作台内存，不会自动创建主线或地图剧情文件。

### 11.6 保存模板

只有当修改适合长期复用时，才点击 **保存模板**。该按钮会直接写回公共模板文件，会影响后续所有同类生成。

保存前确认：

- 模板不是只适用于本次剧情的临时写法。
- 必须桥段具有通用性。
- 关系边界没有被放宽到不合理程度。
- Prompt 已经过至少一轮候选审核。

## 12. 手机 AI 工作台

路径：**内容库 > 手机 AI**。

支持三种模式：

| 模式 | 用途 |
|---|---|
| 自由文字消息 | 测试手机文字聊天回复 |
| AI 通话首句 | 测试接通电话后的第一句 |
| AI 通话后续 | 测试通话中的后续回复 |

### 12.1 离线测试

1. 选择模式。
2. 填写角色、玩家称呼、关系阶段、亲密度、信任度和情感风味。
3. 填写时间、心情、地点、回复节奏、测试记忆和历史。
4. 填写玩家输入和原始响应 JSON。
5. 点击 **构建并审核**。

离线预览不联网，也不写角色存档。

### 12.2 真实请求

点击真实请求按钮会把最终系统 Prompt 和最近历史发送给外部 API，并消耗配额，但不会写入角色记忆或游戏存档。

注意：

- “不写存档”不等于“不联网”。
- 不要在测试记忆、历史或玩家输入中填写密码、Token、密钥或隐私信息。
- 响应使用 `[SPLIT]` 分段。
- 括号动作会从语音台词中移除。
- 空响应、过长回复、无效 `choices[0]` 会显示诊断。

## 13. 系统工具

### 13.1 Schema 迁移

路径：**系统工具 > Schema 迁移**。

该工具用于把旧版剧情数据升级到当前结构。它不是日常内容编辑功能。

操作顺序：

1. 扫描文档版本。
2. 选择“可迁移”的文档。
3. 仔细检查字段变更预览。
4. 确认迁移。
5. 记录界面给出的永久备份路径。

迁移会创建带时间戳的 `.bak`，然后直接写盘并回读验证。如果预览后源文件发生变化，迁移会拒绝继续，必须重新扫描。

不要迁移“未来版本”或“无效 JSON”。遇到异常时使用迁移生成的备份或版本控制恢复。

### 13.2 运行时监视

路径：**系统工具 > 运行时监视**。

1. 从 Godot 编辑器运行游戏。
2. 打开运行时监视。
3. 选择调试 Session。
4. 按事件、来源、剧情 ID 或错误筛选。
5. 查看当前剧情、章节、事件序号和完整数据。

它适合排查：

- 剧情是否真正触发。
- 触发来源是日程、地图、Guide 还是直接加载。
- 运行停在哪个章节和事件。
- 是否出现运行时错误。

“清空当前 Session”只删除监视缓存，不会修改剧情文件。

## 14. 保存、撤销与版本控制

### 14.1 各编辑器的保存规则

| 编辑区域 | 先应用 | 最后保存 | 错误是否阻止保存 |
|---|---:|---:|---:|
| 主线 / 地图 Inspector | 是 | 顶部“保存剧情” | 否，必须人工检查 |
| 手机消息 Inspector | 是 | 手机消息“保存” | 是 |
| 手机消息完成动作 | 是 | 手机消息“保存” | 是 |
| 固定来电信息 | 是 | 固定来电“保存” | 是 |
| 固定来电台词 | 是 | 固定来电“保存” | 是 |
| Event Registry | 是 | “保存 Registry” | 是 |
| Guide Flow | 是 | “保存 Guide” | 是 |
| 入口调度 | 是 | “保存两个数据源” | 是 |
| 新建固定内容 | 不适用 | 确认即写盘 | 创建前校验 ID |
| Schema 迁移 | 不适用 | 确认即写盘 | 迁移前后校验 |
| AI 模板 | 不适用 | “保存模板”直接写盘 | 应人工审核 |

### 14.2 撤销和重做

每个编辑器有独立撤销记录。撤销只针对已经进入内存的操作。

以下内容通常不能通过撤销恢复：

- 表单中尚未点击应用的文字。
- 已确认的新建文件。
- 已执行的 Schema 迁移。
- 已直接保存的公共 AI 模板。

保存后撤销历史可能仍然存在。继续撤销会让文档重新进入未保存状态。

### 14.3 刷新和重新加载

刷新会重新扫描或读取磁盘内容。以下情况下不要刷新：

- Inspector 已应用但尚未保存。
- 手机消息或固定来电有未保存修改。
- 独立工具窗口已经应用修改但尚未保存。

主线和地图剧情切换有未保存保护，但并非所有内嵌工具和独立窗口都提供统一确认。

### 14.4 提交版本控制前

1. 保存所有打开的内容。
2. 清零校验错误。
3. 运行分支模拟。
4. 在游戏中走通关键路径。
5. 检查版本控制差异，只提交预期文件。
6. 不提交临时响应、密钥、测试 Token 或不必要的备份。

## 15. 推荐策划工作流

### 15.1 固定剧情制作

1. 明确剧情 ID、入口和目标角色。
2. 新建主线或地图剧情。
3. 先搭章节和 Choice 分支骨架。
4. 使用组合模板铺设场景演出。
5. 填写对白和资源。
6. 应用 Inspector 修改。
7. 查看自动校验。
8. 运行分支模拟。
9. 保存剧情。
10. 配置日程、地图、Guide 或 Registry 入口。
11. 运行游戏并使用运行时监视联调。
12. 查看覆盖率报告。

### 15.2 手机消息制作

1. 新建手机消息并填写角色。
2. 先建立消息 ID 和顺序。
3. 添加玩家选项和 `next` 分支。
4. 填写正文、图片和语音参数。
5. 配置完成动作。
6. 清零校验错误并保存。
7. 在手机界面完整走通每条回复路径。

### 15.3 固定来电制作

1. 新建固定来电。
2. 应用通话 ID 和角色。
3. 按播放顺序填写台词。
4. 检查 `[SPLIT]` 和舞台说明。
5. 应用台词并保存。
6. 在剧情中添加“语音通话”事件并选择该 ID。
7. 校验引用并在游戏中试听。

### 15.4 AI 内容制作

1. 选择最接近的模板。
2. 明确环境和关系边界。
3. 填写具体的任务、意外、话题和收束。
4. 先离线预览 Prompt 和清洗结果。
5. 小批量真实生成候选。
6. 比较重复度、越界和可演出性。
7. 人工选择、修改和验收。
8. 只有通用改进才保存到公共模板。

## 17. 常见问题与恢复

### 改了字段，但标题没有出现 `*`

通常是尚未点击“应用到当前剧情”“应用消息修改”“应用通话信息”或“应用台词修改”。先应用，再检查未保存状态。

### 点击应用后，磁盘文件没有变化

这是正常的。应用只更新编辑器内存，还需要点击对应的“保存”。

### 目标章节不存在

检查 Choice 和 Jump 的目标章节。可以在 Inspector 重新选择，也可以在画布中重新连接章节入口。

### 出现不可达章节或无出口循环

检查是否有 Jump 提前结束顺序流、Choice 是否遗漏分支、目标章节是否最终能结束。使用“分支模拟”查看具体路径。

### 手机消息无法保存

优先检查重复消息 ID、无效 `next`、空玩家选项、无出口循环和错误的完成动作。

### 固定来电无法保存

固定来电是共享数组。除了当前通话，还要检查其他通话是否存在重复 ID、空台词、无效角色或失效引用。

### JSON 解析失败

根据诊断中的行号修复 JSON，或从版本控制恢复。不要在结构损坏时反复保存覆盖原文件。

### 资源显示“未注册”

说明 JSON 中保留了一个当前资源目录找不到的 ID。确认资源是否改名、删除或尚未导入，再选择有效资源。

### 误操作但尚未保存

立即使用当前编辑器的撤销。不要刷新、重新加载或切换可能覆盖内存的内容。

### 保存失败

检查：

- 文件是否只读。
- 项目目录权限。
- 文件是否被其他程序占用。
- 磁盘空间。
- Git 或同步软件是否正在锁定文件。

普通 JSON 保存使用临时文件替换，失败时会尝试恢复原文件。

### Schema 迁移后异常

使用迁移界面记录的 `.bak` 文件恢复，或通过版本控制恢复迁移前版本。不要继续编辑异常数据扩大差异。

### AI 请求失败

检查 AI 客户端配置、认证、网络和配额；查看请求元数据中的 HTTP 状态。取消失败队列后再重新发起，避免重复并发请求。

### AI 内容使用了 Fallback

Fallback 只保证可运行。调整模板、关系阶段和创意参数后重新生成，并重新进行人工审核。

## 18. 团队约定建议

为减少多人协作冲突，建议项目组统一以下规则：

1. ID 使用小写英文、数字和下划线。
2. 主线 ID 使用 `角色_main_序号`。
3. 地图剧情 ID 使用 `地点_event_主题`。
4. 手机消息 ID 使用 `角色_chat_主题`。
5. 固定来电 ID 使用 `角色_call_主题`。
6. 章节 ID 使用语义名称，不使用临时的 `new_1`、`test_2`。
7. 同一时间尽量只由一人编辑共享固定来电、入口调度和公共 AI 模板。
8. 每次提交只包含一个明确内容任务。
9. 提交前完成校验、模拟和关键路径联调。
10. 高级 JSON 和 Schema 迁移由熟悉协议的人员操作。

剧情编辑器负责减少手工 JSON 错误，但不能代替内容验收。最终判断标准始终是：入口能触发、流程能走完、演出符合预期、分支结果正确、AI 内容处于角色与关系边界内。
