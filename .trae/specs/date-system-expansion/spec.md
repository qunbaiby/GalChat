# 约会系统拓展 Spec

## Why
目前的约会系统相对简单，缺乏阶段性的规划与时间维度的结合。通过引入“早、中、晚”三阶段的约会槽位，并结合具体的约会类型与地点数据，可以为玩家提供更具策略性和沉浸感的约会体验。

## What Changes
- **UI 布局调整**：将底部栏的“放弃”按钮移动到场景左上角，并修改为“返回”样式（带有左箭头图标）。
- **约会槽位增加**：在“去约会”按钮左侧增加三个槽位，分别对应“早上”、“下午”、“晚上”。
- **新增约会数据源**：构建 `date_config.json`，定义“漫步散心”、“逛街购物”、“观影看展”、“餐饮小聚”四种约会类型，并关联地图数据 `map_data.json` 中的具体地点。
- **下拉列表与添加逻辑**：在右侧面板中，展示约会类型。点击类型展开包含对应地点的子列表。地点项包含“+”按钮，点击后将地点添加至可用的时间槽位中。点击槽位中已添加的地点可以将其撤销。
- **时间限制逻辑**：读取游戏当前时间，如果当前已经是下午，则早上的槽位将被禁用/占位；如果是晚上，则早上和下午的槽位均被禁用。
- **接口预留**：点击“去约会”时，收集当前槽位中的约会计划数据，并调用预留的 `_start_date_plan()` 接口，暂不实现具体的约会流程。

## Impact
- Affected specs: 约会系统的 UI 交互、时间系统的状态读取。
- Affected code:
  - `scenes/ui/date/date_scene.tscn`
  - `scripts/ui/date/date_scene.gd`
  - 新增 `assets/data/interaction/date_config.json`

## ADDED Requirements
### Requirement: 约会阶段槽位与时间校验
系统 SHALL 提供三个约会槽位（早、中、晚），并根据当前游戏时间禁用已过期的槽位。

#### Scenario: 下午安排约会
- **WHEN** 玩家在游戏时间为下午时进入约会界面
- **THEN** “早上”的槽位显示为不可用（置灰或显示已被占用），玩家只能在“下午”和“晚上”的槽位中添加约会地点。

### Requirement: 约会地点选择与撤销
系统 SHALL 允许玩家从展开的地点列表中点击“+”号添加约会，并允许点击槽位撤销。

#### Scenario: 添加与撤销约会
- **WHEN** 玩家点击地点列表的“+”按钮
- **THEN** 该地点被添加至第一个可用的时间槽位中。
- **WHEN** 玩家点击已被填入地点的槽位
- **THEN** 该地点从槽位中移除，槽位重新变为空闲状态。
